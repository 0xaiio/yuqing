# SCRFD + AdaFace 实施 README

这个文件专门记录“将现有人脸检测/人物识别链路升级为 `SCRFD + AdaFace`”的实施计划、约束和完成状态。

## 目标

将当前基于 OpenCV 手工特征的人脸识别流程，升级为：

- `SCRFD`：负责人脸检测和 5 点关键点
- `AdaFace`：负责人脸 embedding
- 保持现有人物库、人物参考图、按人物图检索、人脸簇绑定、自然语言中使用人名的业务能力不变

## 设计原则

1. 优先本地部署，首次运行自动下载模型
2. 不破坏现有前后端接口
3. 老数据可自动迁移或自动重建索引
4. 对 CPU 可运行场景友好
5. 模型和阈值全部配置化

## 模型方案

### 检测模型

- 默认：`buffalo_sc / det_500m.onnx`
- 备选：`buffalo_l / det_10g.onnx`

当前实现里默认选了 `buffalo_sc`，原因是：

- 首次下载更轻，适合桌面本地部署
- 仍然是 `SCRFD` 检测链路
- 后续可以通过配置切到 `buffalo_l`

### 识别模型

- 默认：`minchul/cvlface_adaface_ir50_webface4m`
- 备选：`minchul/cvlface_adaface_ir101_webface4m`

推荐默认 `IR50` 的原因：

- 体积和推理延迟更适合桌面本地部署
- 比 `IR101` 更容易作为默认配置放进一键启动链路
- 后续可以通过配置切换到 `IR101`

## 已知约束

- `InsightFace` 开源代码可用，但其模型使用边界需要注意
- `AdaFace` 模型来自官方 Hugging Face 仓库，首次下载耗时会明显高于当前纯 OpenCV 方案
- 本地 CPU 可运行，但首次索引重建会比原方案更慢
