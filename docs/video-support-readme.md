# 视频支持实施 README

这个文件专门记录“将当前跨源图片智能管理系统扩展到同样支持视频”的实施计划、模型选型和完成状态。

## 目标

把当前仅支持图片的导入、分析、检索链路扩展到视频，覆盖：

- 监听并导入视频文件
- 提取视频缩略图、时长、分辨率、帧率等元数据
- 检测并识别视频中的人物
- 提取视频中的场景与内容摘要
- 支持基于文本和视频样例的视频检索

## 模型方案

### 1. 人物检测与识别

- `SCRFD`：逐帧人脸检测
- `AdaFace`：逐帧人脸 embedding

理由：

- 当前工程已经稳定接入这套链路
- 对“已知人物库 + 本地识别”场景非常契合
- 扩展到视频时，只需要把单图识别扩展到“关键帧集合 + 时序聚合”

### 2. 视频场景理解

- 本地语义检索向量：`SigLIP2`
- 多帧摘要与标签：`Qwen2.5-VL`

理由：

- `SigLIP2` 适合做图文检索向量，便于把文本查询和关键帧映射到同一向量空间
- `Qwen2.5-VL` 官方明确支持长视频理解与事件定位，适合多帧摘要、场景标签和检索辅助标签生成
- 对当前工程来说，最实用的落地方式是“关键帧抽样 + 多帧理解”，兼顾本地部署和 API 兼容性

### 3. 视频切分与抽帧

- 第一阶段：均匀抽帧 + 首尾帧保底
- 第二阶段：接入 `TransNetV2` 做镜头边界检测

理由：

- 第一阶段先把视频支持完整打通，保证工程闭环
- 第二阶段再升级镜头级抽帧与镜头级聚合，进一步提升视频语义和检索精度
